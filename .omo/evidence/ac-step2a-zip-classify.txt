Key regression checks proven by tests:
- 연결감사보고서 is never double-classified as plain 감사보고서 (mutually exclusive).
- A 사업보고서 ZIP with only the main XML (audit inline, no separate entry) yields
  audit_entries == [] / consolidated_audit_entries == [] as a legitimate not-found
  state, not a DocumentZipError.
- Corrupt bytes and empty bytes both return typed DocumentZipError (no traceback).
- DOCUMENT-NAME within the ~8KB head chunk is still parsed correctly even when the
  XML entry is padded to > 8KB total size (proves head-read, not full-file read).
